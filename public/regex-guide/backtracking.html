Catastrophic Backtracking

Die meisten Regex-Engines können in einen Zustand fallen, in dem ein einzelner Match-Versuch Stunden dauert. Auf einem 30-Zeichen-String. Das nennt man Catastrophic Backtracking oder, wenn ein Angreifer es ausnutzt, ReDoS (Regular Expression Denial of Service).

Was passiert da überhaupt?

NFA-Engines (PCRE, JS, Python, Java, .NET, ...) probieren Pfade durch das Muster und gehen zurück, wenn ein Pfad nicht klappt. Das nennt man Backtracking. Normalerweise harmlos.

Aber: wenn ein Muster mehrere überlappende Wege bietet, dasselbe zu matchen, wächst die Zahl der zu probierenden Kombinationen exponentiell mit der Eingabelänge. Der Klassiker:

Muster:  (a+)+b
Eingabe: aaaaaaaaaaaaaaaaaaaaaaaaaaaaa    (29× a, kein b)

Die Engine versucht: nimmt a+ alle 29 a's. Kein b da. Backtrack. Versucht 28 + 1, 27 + 2, 27 + 1 + 1, 26 + 3, ... Es gibt 2^28 Aufteilungen der 29 a's in zwei a+-Gruppen. Die Engine probiert sie ALLE, bis sie aufgibt.

Auf meinem Laptop: 30 a's = 5 Sekunden. 35 a's = 3 Minuten. 40 a's = unbenutzbar. Bei 50 a's könnte die Sonne erkalten, bevor das Match fehlschlägt.

Die roten Flaggen — erkennt sie

Drei Muster, die FAST IMMER explodieren:

  1. Quantor in Quantor: (a+)+, (a*)*, (a+)*. Verschachtelte Quantoren auf demselben Token. tödlich
  2. Alternativen mit Überlappung: (a|aa)+, (\w|\d)+. Beide Alternativen können dieselben Zeichen schlucken — exponentielle Wege.
  3. Greedy + verschachtelt: ".*"|"[^"]*" mit einer langen un-terminerten Eingabe — die Engine probiert sich exponentiell tot.

Beispiel aus der Realität — Cloudflare 2019

Cloudflare hatte 2019 eine 27-minütige globale Outage durch eine einzige Regex in einem WAF-Regelsatz. Das Muster sah ungefähr so aus:

(?:(?:\"|'|\]|\}|\\|\d|(?:nan|infinity|true|false|null|undefined|symbol|math)|
\`|\-|\+)+[)]*;?((?:\s|-|~|!|\{\}|\|\||\+)*.*(?:.*=.*)))

Sieht plausibel aus. War katastrophal. Eine bestimmte Eingabe ließ es exponentiell explodieren — und Cloudflare lief auf 100% CPU. Komplette Edge ausgefallen.

Moral: jede Regex, die du auf User-Input loslässt, ist ein potentieller DoS-Vektor.

Wie du dich schützt

Strategie 1: vermeide Quantor-in-Quantor

Wenn du (\w+)+ hast, was wolltest du wirklich? Meistens kannst du den inneren Quantor weglassen:

(\w+)+        → \w+         meistens äquivalent
(a+)+         → a+
([a-z]*)*     → [a-z]*

Strategie 2: spezifischere Patterns

Statt ".*" für Strings in Anführungszeichen:

".*"          ← gefährlich bei langen Eingaben ohne schließenden Quote
"[^"]*"       ← schnell, weil non-greedy by class
"(?:[^"\\]|\\.)*"   ← korrekt mit Escape-Sequenzen, aber stabil

Die letzte Form heißt „unrolled loop" und ist linear schnell, weil sie überlappende Pfade ausschließt.

Strategie 3: atomare Gruppen + possessive Quantoren

Erinnerst du dich an den Magier? (?>...) und *+ erlauben kein Backtracking. Wenn du sie einsetzt, ist die Explosion strukturell unmöglich:

(a+)+b              ← katastrophal
(?>a+)+b            ← linear, kein Backtracking ins innere a+
(a++)+b             ← dasselbe per possessive Quantor (PCRE/.NET)

Strategie 4: nimm eine RE2-Engine

Go's regexp, Rust's regex-Crate, Hyperscan — alle garantiert linear. Du kannst kein Catastrophic Backtracking auslösen, weil sie keinen Backtracker haben.

Trade-off: kein Lookbehind, keine Backreferences, keine Recursion. Für Web-Server-Filter ein guter Trade.

Strategie 5: Timeouts

Wenn dein Stack das erlaubt, setz einen Timeout auf jede Regex-Operation. .NET, Java, modernes Python — viele Engines unterstützen das. Vor Timeout ist verlorene Zeit, aber kein verlorener Server.

// .NET
var rx = new Regex(pattern, RegexOptions.None, TimeSpan.FromMilliseconds(100));

Test deine Pattern

  • regex101.com — zeigt die Anzahl der Schritte. Wenn du bei moderaten Eingaben >10000 siehst, ist was faul.
  • safe-regex (npm-Paket) — heuristische Erkennung von ReDoS-Risiken.
  • recheck (Cybozu) — moderner ReDoS-Detector.
  • Selbst-Test: ein langer String aus dem „bösen" Zeichen, ohne den abschließenden „guten" Zeichen — wenn dein Regex hängt: Bingo, Bug gefunden.
🚨 Predigt-Schluss Wenn du Regex auf User-Input ausführst — Login-Formular, API, Log-Filter, Markdown-Renderer — und keine Timeouts oder eine RE2-Engine hast, ist es eine Frage der Zeit, bis dich jemand abschießt.
