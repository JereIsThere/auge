Dialekte

„Regex" ist kein einheitlicher Standard. Es gibt grobe Familien, und innerhalb jeder Familie eigene Akzente. Wer mehr als eine Sprache kennt, läuft permanent in „aber das ging doch in PCRE"-Fallen. Wir gehen die wichtigsten Unterschiede durch.

Die zwei Familien

FamilieEnginesEigenheit
NFA-basiert (mit Backtracking)PCRE, JS, Python, Ruby, Java, .NET, PerlLookarounds, Backreferences, ALLES — aber kann katastrophal langsam werden
DFA-basiert (linear)Go's RE2, Rust's regex-Crate, Hyperscan, awkGarantiert linear in der Laufzeit. Kein Backtracking, daher KEINE Backreferences und KEINE Lookarounds

Das ist der große Bruch. Wenn du in Go programmierst und denkst „ich nutze einfach Lookahead", gute Nachricht: kannst du nicht. Schlechte: du denkst um.

Die wichtigsten Akzent-Stolperfallen

Python: re vs regex

Das Python-Standard-Modul re kann eine Menge nicht. Das externe regex-Modul (drop-in compatible) ist mächtiger:

  • Lookbehind variable Länge: nur regex.
  • Possessive Quantoren: nur regex.
  • Bessere Unicode-Properties: nur regex.
pip install regex
import regex as re   # statt: import re

JavaScript: das u- und v-Flag

Ohne u arbeitet JS-Regex auf UTF-16 Code-Units, nicht auf Codepoints — Emojis brechen. Mit v-Flag (ES2024+) kommen Set-Operationen in Klassen dazu: [\p{L}--\p{Lu}] = „Buchstaben außer Großbuchstaben".

Go (RE2): wirklich kein Lookahead

Es gibt keinen Trick, das nachzubauen. Wenn du Lookarounds brauchst, lade regexp/syntax raus und nimm eine alternative Lib (github.com/dlclark/regexp2 portiert .NET-Regex).

sed: BRE vs ERE

Klassisches sed nutzt BRE (Basic Regular Expressions). Da müssen (, ), +, ?, { mit Backslash escaped werden — sonst sind's Literale. Mit sed -E oder sed -r kriegst du ERE, das deinem Bauchgefühl näher ist.

sed 's/(foo)/\1!/'           # nichts passiert in BRE
sed 's/\(foo\)/\1!/'         # so geht's in BRE
sed -E 's/(foo)/\1!/'        # ERE, sieht aus wie überall sonst

grep: drei Modi

  • grep = BRE — Klammern müssen escaped
  • grep -E = ERE — modern
  • grep -P = PCRE — wie in Perl/Python (nicht überall verfügbar — z.B. macOS-Default-grep hat das nicht)

vim: eigene Welt

Vim hat fünf Magie-Levels (\v, \m, \M, \V). Standard ist „magic": die meisten Metazeichen sind Literale. Mit \v („very magic") wird's wie überall sonst:

:%s/\v(\w+)\s+\1/\1/g     # mit very-magic, lesbar

Empfehlung: was sollst du lernen?

  1. Lerne PCRE-Style als Default. Das ist die Lingua Franca: PCRE, Python, JS, Java, .NET sind 90% kompatibel zueinander.
  2. Wenn du in Go/Rust schreibst, akzeptiere die DFA-Beschränkungen. Sie sind ein Feature: keine ReDoS-Sorgen.
  3. In Bash/sed/awk-Land: nimm grep -E oder perl -pe für Komplexes — viel weniger Backslash-Hölle.
