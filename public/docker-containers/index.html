💡 Multi-stage Dockerfile

# Build Stage (enthält Dev-Tools)
FROM python:3.12 AS builder
WORKDIR /app
COPY requirements.txt .
RUN pip install --prefix=/install -r requirements.txt

# Final Stage (nur Runtime)
FROM python:3.12-slim
COPY --from=builder /install /usr/local
COPY . .
CMD ["python", "app.py"]
# Resultat: deutlich kleineres Image!

💡 K8s Grundstruktur

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
        - name: app
          image: my-app:1.0
          ports:
            - containerPort: 8000

Security Best Practices

  • Nie als root laufen (USER 1000 im Dockerfile)
  • Read-only Filesystem wo möglich
  • Capabilities droppen (--cap-drop ALL)
  • Distroless Images nutzen (kein bash, kein sh)
