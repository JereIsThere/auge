Windows ↔ Android

Beide Richtungen funktionieren — aber jede hat ihre eigene Falle. Hier sind die Setups, mit denen es wirklich reibungslos läuft.

A. Android steuert Windows

Der einfachere Fall — Windows ist großzügig mit Remote-Input.

Setup auf dem Windows-PC

  1. RustDesk als Service installieren (siehe Grundlagen).
  2. Permanent Password setzen.
  3. ID notieren.

Setup auf dem Android-Gerät

  1. RustDesk-App installieren (Play Store oder APK).
  2. App öffnen → ID des PCs eingeben → Connect → Permanent Password.

Bedien-Modi auf Android

Oben im Verbindungs-Bildschirm gibt's einen Toggle zwischen zwei Modi — kennt fast keiner, macht aber den Unterschied:

ModusWie es sich anfühltWofür
Touch / DirectWo du tippst, wird geklicktSchnell mal was klicken, schlechte Präzision bei kleinen Targets
Mouse / TrackpadTouch-Display als Trackpad — Cursor bewegt sich relativPräzise Arbeit, kleine Buttons, längere Sessions

Im Trackpad-Modus zählen die typischen Gesten:

  • 1-Finger-Tap = Linksklick
  • 2-Finger-Tap = Rechtsklick
  • 2-Finger-Drag = Scrollen
  • Long-Press + Drag = Drag & Drop
💡 Unorthodoxer Tipp Schließ eine OTG-Maus oder Bluetooth-Tastatur an dein Android — RustDesk leitet die Eingaben durch. Dann ist dein Telefon ein vollwertiger Mini-Laptop für SSH ins Heimnetz oder eine Notfall-RDP-Session. Mit einem 7"-Tablet + Mini-Maus passt das in jede Tasche.

On-Screen-Keyboard für Sondertasten

In der Tab-Bar unten gibt's drei Keyboard-Modi:

  • Soft — die normale Android-Tastatur. Für reines Tippen.
  • Translate — wandelt deine Eingaben in „echte" Keystrokes um. Wichtig für Spiele, Terminals, Sondertasten.
  • Legacy / Map — Roh-Keystrokes, jede Taste 1:1. Für komplexe Shortcuts.

Für eine deutsche Tastatur am PC + Android: Translate-Modus benutzen, sonst kommt aus „ä" plötzlich „;". Mehr dazu im Troubleshooting.

B. Windows steuert Android

Der knifflige Fall — Android lässt aus Sicherheitsgründen nicht jede App Input simulieren oder Bildschirm aufnehmen.

Setup auf Android (das Gesteuerte)

  1. RustDesk aus Play Store / APK installieren.
  2. RustDesk Input als separate App installieren — dieselbe Quelle.
    Wichtig Ohne diese App kannst du das Gerät nur anschauen, nicht steuern.
  3. RustDesk öffnen → "Start Service" tippen → Permission-Dialoge:
    • Display over other apps (für Input-App) — erlauben.
    • Accessibility Service für RustDesk Input — manuell in den Android-Settings aktivieren (RustDesk öffnet die Einstellungen für dich).
    • MediaProjection — beim ersten Connect kommt der Dialog „Aufnahme starten" → erlauben.
    • Notification für Persistent-Service.
  4. One-Time-Code wird angezeigt oder Permanent Password setzen.

Setup auf Windows (das Steuernde)

  1. RustDesk öffnen → ID des Android-Geräts eingeben → Connect.
  2. Code/Passwort eingeben.
  3. Du siehst den Android-Bildschirm in einem Fenster.

Spezielle Gesten / Tasten von Windows aus

Der Toolbar oben in der Verbindung hat Android-spezifische Buttons:

  • Home / Back / Recent Apps — die drei Android-Navi-Buttons als Klick-Buttons
  • Volume +/–, Power — Hardware-Tasten simuliert
  • Rotation lock — sinnvoll, sonst dreht sich das Bild auf dem PC, wenn du das Telefon kippst

Maus + Tastatur am PC funktionieren nativ — Cursor wird auf Android übertragen, Tastendrücke kommen als Hardware-Events durch.

⚠️ Android-Gotcha: Service stoppt nach Reboot Nach einem Neustart ist der RustDesk-Service auf vielen Android-Skins (Xiaomi, Samsung, OnePlus) nicht automatisch aktiv. Workaround: in den Android-Settings RustDesk + RustDesk Input von der Akku-Optimierung ausnehmen UND als Autostart-App freigeben (auf Xiaomi: "Berechtigungen" → "Autostart"). Sonst musst du nach jedem Reboot manuell starten.
💡 Unorthodoxer Workaround Wenn dein Android nach Reboot nicht selbst startet, hau eine Tasker- oder MacroDroid-Regel rein: „Bei Boot abgeschlossen → öffne RustDesk → tippe ‚Start Service'". Funktioniert zuverlässiger als die offizielle Autostart-Geschichte und überlebt System-Updates.

Was nicht geht

  • DRM-geschützter Inhalt (Netflix, Disney+) erscheint schwarz — MediaProjection-Limit, kein RustDesk-Bug.
  • Banking-Apps blocken oft Screen-Capture per FLAG_SECURE. Workaround: ROOT (auf eigene Gefahr) oder die App auf einem Alt-Gerät bedienen.
  • Sehr alte Android-Versionen (< 7) haben kein MediaProjection — RustDesk geht erst ab Android 7+ richtig.

C. Beide Wege gleichzeitig

Du kannst auf beiden Geräten gleichzeitig steuernd UND steuerbar sein. Praktisches Setup:

  • Auf Windows + Android beide Geräte als „verfügbar" einrichten (Permanent Password, Service läuft).
  • Address Book auf beiden — gegenseitig die ID des anderen abspeichern.
  • Vom Telefon → PC starten, oder vom PC → Telefon starten — je nach Bedarf.

Praxis-Use-Case: Du sitzt am PC, willst eine 2FA-SMS aus deinem Android holen → vom PC aus auf Android verbinden, SMS-Code lesen, fertig. Brauchst nicht zum Telefon greifen.

D. Android steuert Android

Ja, geht. Beide Geräte: RustDesk + RustDesk Input + alle Permissions wie oben. Use-Case: Familie helfen, ohne dass sie den PC starten müssen.

Auf dem steuernden Android: Connect zum anderen Android. Die Toolbar oben enthält dann beide Sets von Buttons (Sender + Empfänger Hardware-Keys). Etwas eng auf kleinen Displays — Tablet ist hier deutlich angenehmer.
