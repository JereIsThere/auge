CLI-Flags (Client)

rustdesk --connect <ID>                # direkt verbinden
rustdesk --connect <ID> --password X    # mit Passwort
rustdesk --file-transfer <ID>          # nur Datei-Transfer
rustdesk --port-forward <ID>           # nur TCP-Tunnel
rustdesk --install-service             # Service installieren (Win)
rustdesk --uninstall-service
rustdesk --server                      # nur als Server starten (kein UI)
rustdesk --import-config FILE          # Config aus Datei importieren

CLI-Flags (Server hbbs)

hbbs -r relay.example.com:21117    # Relay-Hostname mitteilen
hbbs -k _                          # nur Verbindungen mit registriertem Key
hbbs -k YOURKEY                    # Custom Key
hbbs --port 21116                  # Port ändern (nicht empfohlen)
hbbs --rmem 0                      # UDP-Receive-Buffer (für sehr aktive Server)

Reset-Routinen

  • Zoom hängt → Display → View Style → Shrink (oder Verbindung neu).
  • Tastatur-Layout falsch → Keyboard → Map probieren, dann Translate.
  • Ruckler → Codec wechseln + Image Quality auf Adaptive.
  • Service tot → systemctl restart rustdesk (Linux) / Service-Manager (Win).
  • Komplette Settings nuken → ~/.config/rustdesk/ löschen, neu starten.

Verbindungs-Info ablesen

Im Verbindungsfenster oben links auf das ⓘ-Symbol → Box zeigt:

  • Connection Type: P2P (direkt) oder Relay (über Server)
  • Codec: aktiver Codec + ob HW-encoded
  • Latency: Roundtrip in ms
  • Bandwidth: aktuelle Datenrate
  • FPS: gerenderte Frames pro Sekunde

Faustregeln:

  • Latency < 30 ms = lokales Netz, Direct.
  • Latency 50–150 ms = WAN-Direkt, gut bedienbar.
  • Latency > 200 ms = wahrscheinlich Relay, Bedienung wird zäh.
