Quick Access — der versteckte Launcher

Drück Ctrl+Shift+Space (Win/Linux) oder ⌘+Shift+Space (Mac), egal wo du gerade bist. Es öffnet eine Spotlight-artige Such-Leiste. Tippen → Enter → Inhalt landet im Clipboard / wird in das fokussierte Feld eingefügt.

Modifiers verändern die Aktion:

  • Enter → Default-Aktion (meistens „autofill" oder „in Clipboard").
  • ⌘+Enter → Item öffnen.
  • ⌘+C auf einem Treffer → Passwort kopieren.
  • Pfeil-rechts → alle Felder durchblättern und einzeln kopieren.
💡 Lifechanger Wenn du Quick Access internalisierst, brauchst du die 1Password-App selber fast nie wieder zu öffnen. Lerne diesen einen Shortcut.

Watchtower richtig lesen

Watchtower ist die Sicherheits-Übersicht über deinen Vault. Findest du in der Sidebar unter „Watchtower". Was die Kategorien wirklich bedeuten:

Travel Mode

Markier Vaults als safe for travel. An der Grenze schaltest du Travel Mode ein → alle nicht-safe Vaults werden vom Gerät entfernt. Nicht versteckt — komplett weg. Nicht mal forensisch wiederherstellbar.

Aktivieren auf my.1password.com → Profile → Travel Mode. Pro Vault setzt du dort, ob er „safe for travel" ist.

Use-Case Reisen in Länder, bei denen Grenzbeamte die Entsperrung von Geräten verlangen können. Sie sehen nur, was als safe markiert ist. Nach Ankunft am Ziel: Travel Mode aus → Vaults syncen zurück.

Custom Password Recipes pro Item

Nicht alle Sites akzeptieren das Default-Recipe. Bank-Logins limitieren oft auf 16 Zeichen, manche verbieten Sonderzeichen, andere erlauben nur a-z+0-9.

Fix: beim Generieren auf das Recipe-Symbol klicken → Länge / Charset einstellen → "Save recipe with item". 1Password merkt sich pro Item, welches Recipe du beim nächsten Rotieren willst.

PIN/numerisch

Es gibt einen "PIN"-Modus für reine Zahlen — gut für Bank-PINs, Smart-Lock-Codes, Telefon-PINs. Setz das Item auf "Password" (nicht Login) → Recipe → Numbers only.

Items mit Nicht-1Password-Nutzern teilen (Item-Sharing-Links / „psst")

Du willst einer Person ein Passwort schicken, die kein 1Password hat? Niemals via Slack/E-Mail/SMS. Stattdessen:

  1. Item öffnen → "Share" → "Get a link to share".
  2. Wähle: Verfallszeit (1h, 1d, 7d, 14d, 30d), maximale Aufrufe (1 / unlimited), nur an bestimmte E-Mail.
  3. Du bekommst einen Link von share.1password.com. Den schickst du.

Die Empfänger-Person sieht das Item einmal, kann es kopieren — danach ist es weg. End-to-End-verschlüsselt; 1Password selbst sieht den Inhalt nicht.

⚠️ Trotzdem kein Allheilmittel Der Link landet in irgendeinem Chat-Verlauf. Wer den Chat liest bevor der Empfänger den Link öffnet, klaut den Inhalt. Setz „nur 1 Aufruf" und schick den Link über einen anderen Kanal als das Passwort selber gehörte (z.B. Passwort-Link via Slack, Empfänger-E-Mail-Constraint via SMS).

Smart-Search-Operatoren

In der Suchleiste kannst du Operatoren verwenden:

github                  # Volltext
in:work                 # Nur Work-Vault
tag:dev                 # Nur Items mit Tag „dev"
type:login              # Nur Logins (auch: note, card, identity, ...)
fave:true               # Favoriten
trash:true              # Im Papierkorb
url:atlassian.net       # In URLs suchen
recent                  # Zuletzt benutzt
weak                    # Watchtower-Filter inline
reused
expiring

Operatoren kombinieren: type:login tag:work weak → schwache Work-Logins.

Autofill-Power-Moves

Inline-Menü erzwingen

Wenn 1Password im Browser keinen Vorschlag macht, klick ins Feld und drück ⌘+Shift+X (Mac) / Ctrl+Shift+X (Win) — die Browser-Extension öffnet die Item-Auswahl genau dort.

Mehrere Identities, eine Form

Beim Checkout brauchst du oft Identity (Adresse) + Credit Card + Login. 1Password fragt sequenziell ab — du kannst pro Feld-Gruppe eine eigene Quelle wählen.

Universal Autofill (macOS, native Apps)

Auch in nativen Apps (z.B. Adobe-Login, Steam, IDEs) Autofill nutzen: ⌘+\ auf einem Login-Feld → 1Password autofill, ohne Browser. Funktioniert für alle Apps mit Standard-Textfeldern.

YubiKey als zweiter Faktor für 1Password selbst

Du hast Master Password + Secret Key. Das ist schon stark. Wer einen Schritt weiter will, fügt einen Hardware-Key hinzu:

  1. my.1password.com → Sign-In & Recovery → 2-Step Verification → Add Security Key.
  2. YubiKey einstecken, Touch.
  3. Mindestens zwei Keys registrieren — einer als Backup im Schließfach.

Bei Login auf neuem Gerät: Master Password + Secret Key + YubiKey-Touch.

Archiv statt Löschen

Items, die du nicht mehr brauchst, aber später vielleicht historisch nachschlagen willst (alter Provider, ehemaliger Job): nicht löschen, sondern archivieren. Item öffnen → "..." → "Archive".

Archivierte Items tauchen nicht mehr in normaler Suche auf, sind aber via archived:true-Filter wiederfindbar. Watchtower ignoriert sie. Items im Trash werden nach 30 Tagen gelöscht — Archivierte bleiben für immer.
