Kontextfenster

Das Kontextfenster ist der Arbeitsspeicher des Modells. Alles, was es „sieht" — System-Prompt, History, dein aktueller Input — muss reinpassen. Was rausfällt, ist weg.

1 Million Token klingt nach „passt alles". 1M Token ≈ 750.000 Wörter ≈ 1500 Buchseiten. Praktisch passen fünf bis zehn dicke Bücher ins Fenster.

Drei Konstruktionsregeln

Regel 1: Kosten skalieren mit Eingabelänge

API-Anbieter rechnen pro Token. Bei 200k Token Input zahlst du 200k mal Input-Preis. Eine Konversation mit dauernder History kann pro Turn schnell ein paar Cent kosten — bei 100 Usern × 100 Turns = Geld.

Regel 2: Long Context ≠ perfektes Memory

Auch wenn das Fenster groß ist, leiden viele Modelle an "lost in the middle": Information am Anfang oder Ende wird zuverlässig genutzt, in der Mitte vergessen. Faustregel: kritische Infos an Anfang oder Ende des Prompts platzieren.

Regel 3: Token zählen ist Pflicht

Wer Long-Context nutzt, muss vor jedem Call wissen, wie viele Token er sendet. tiktoken oder das Anbieter-eigene Tool. Passiert sonst: API-Error mid-flight, schlechte Antwort, oder horrende Rechnung.

Tragwerksprobleme & Lösungen

Problem: Conversation wächst über die Zeit

Lösungen, hierarchisch:

  • Sliding Window — nur die letzten N Turns mitnehmen. Einfach, aber Kontext geht verloren.
  • Summarization — Älteres zu einer Zusammenfassung verdichten. Verbraucht zusätzliche Calls.
  • Memory-Layer — wichtige Fakten in eine externe DB ziehen, bei Bedarf via RAG abrufen.
  • Prompt-Caching — Anbieter wie Anthropic cachen statische Prompt-Teile, billiger pro Call. Nutzen.

Problem: Riesendokument analysieren

Don't: 200k-Token-Dokument auf einmal reinwerfen und „alles wichtige zusammenfassen" sagen. Modell wird Mittelteil wegmitteln.

Do: in semantische Chunks splitten (Kapitel, Sections), pro Chunk separat zusammenfassen, dann die Zusammenfassungen zusammensetzen. Map-Reduce für LLMs.

Die unsichtbare Grenze: Output-Tokens

Selbst wenn das Eingabe-Fenster 200k beträgt, ist die Output-Generierung oft kürzer limitiert (4k–32k). Wenn du eine 50.000-Wort-Übersetzung willst, schaff es nicht in einem Call. Streamen, chunken, wieder zusammenfügen.
