Tokenisierung

Modelle lesen keine Buchstaben. Sie lesen Zahlen. Tokenisierung ist der Mapping-Schritt: Text → Liste von Integern. Klingt trivial. Ist es nicht.

Definition

Sei V ein endliches Vokabular von Strings (typisch: 30k–200k Einträge). Ein Tokenizer ist eine Funktion T: String → [Int], die einen Eingabestring in eine Sequenz von Indizes in V abbildet. Umkehrfunktion T⁻¹ existiert (verlustfrei für die meisten modernen Tokenizer).

Beispiel — exakte Werte für GPT-4-Tokenizer (cl100k)

"Hallo Welt"        → [9906, 23748]               (2 Tokens)
"Erdbeere"          → [36, 6634, 12, 90716]        (4 Tokens, deutsche Umlaute teurer)
"strawberry"        → [496, 675, 15717]            (3 Tokens)
"🍓"                → [9468, 235, 241]             (3 Tokens — Emoji = mehrere Bytes)
"AAAAAAAAAAAA"      → [32, 32, 32, ...]            (oft 1 Token pro A, weil ungewöhnlich)
"the"               → [1820]                       (1 Token, sehr häufig)

Drei Konsequenzen, die in der Praxis zählen

Lemma 1: Deutsche Texte sind teurer als englische

Das Vokabular wird auf englischen Daten trainiert. Folge: deutsche Wörter zerfallen in mehr Tokens. „Donaudampfschifffahrtsgesellschaftskapitän" sind ~14 Tokens. Das englische Equivalent gibt es nicht — was vielleicht ein Geschenk ist.

Lemma 2: Modelle sehen keine Zeichen

„Wie viele R sind in 'Strawberry'?" ist für ein LLM kein Buchstaben-Zähl-Problem. Es sieht [496, 675, 15717]. Buchstaben existieren nicht in seinem Universum, außer es hat das Wort buchstabenweise auswendig gelernt. Daher die berühmten Counting-Fails.

Lemma 3: Whitespace ist wesentlich

Im BPE-Tokenizer ist " Welt" (mit Leerzeichen) ein anderes Token als "Welt" (ohne). Das Modell sieht eine Welt als eine Welt-mit-Leerzeichen oder ohne — und behandelt sie verschieden. Ein einsamer Trailing-Space im Prompt kann die Antwort sichtbar verändern.

Korollar: was du ablesen kannst

  • Kostenrechnung: Englische API-Kosten ≠ deutsche API-Kosten für „denselben" Text. Differenzfaktor 1.4–2×.
  • Tokenizer-Tools: tiktoken für OpenAI, tokenizers (Hugging Face) für alles andere. Nutze sie zum Schätzen vor einem teuren API-Call.
  • Konsistenz: derselbe String → exakt dieselbe Token-Liste, deterministisch. Aber nicht über Tokenizer-Versionen hinweg — Models & Tokenizer kommen als Paar.
