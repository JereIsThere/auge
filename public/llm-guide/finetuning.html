Fine-Tuning

Fine-Tuning ist, wenn du dem Modell einen spezifischen Trick beibringst, indem du ihm Beispiele zeigst — nicht im Prompt, sondern in den Gewichten. Nach dem Training kann es das Verhalten ohne Beispiele im Prompt reproduzieren.

Erst die Wahrheit: brauchst du Fine-Tuning überhaupt?

Wahrscheinlich nicht. Erst diese Reihenfolge probieren:

  1. Besseres Prompting (Few-Shot, klare Struktur).
  2. Größeres Modell (GPT-4 statt 3.5).
  3. RAG für fehlendes Wissen.
  4. Tools für fehlende Fähigkeiten.
  5. DANN — wenn Stil/Format/Verhalten konsistent gemacht werden muss — Fine-Tuning.

Wer mit Fine-Tuning anfängt, weil Prompts zu lang werden — Disziplin, Sportler. Erst die einfachen Wege.

Wofür Fine-Tuning gut ist

  • Stil-Konsistenz — dein Modell soll immer in deinem Brand-Tone schreiben.
  • Format-Konsistenz — JSON-Output mit deinem genauen Schema, ohne Slips.
  • Domain-Sprache — medizinische, juristische, interne Fachsprache, die dem Basismodell schwerfällt.
  • Token-Effizienz — Few-Shot-Beispiele können aus dem Prompt verschwinden, weil das Modell sie internalisiert hat.
  • Distillation — kleines Modell zum Verhalten eines großen trainieren, billiger im Betrieb.

Wofür es NICHT gut ist

  • Wissen einbauen, das du täglich aktualisierst. Nutz RAG. Fine-Tuning ist langsam, sperrig, einmalig.
  • Halluzinationen reduzieren. Tut Fine-Tuning kaum. RAG > FT.
  • Fähigkeiten beibringen, die das Basismodell nicht hat. Du machst keinen GPT-3.5 zu GPT-4 mit FT.

Die Methoden — leicht bis schwer

1. Full Fine-Tuning

Alle Gewichte werden trainiert. Maximale Kontrolle, maximaler GPU-Hunger, maximaler Disk-Verbrauch (du kriegst eine eigene Modellkopie). Bei großen Modellen: für Mortals nicht praktikabel.

2. LoRA / QLoRA — der echte Champion

Statt alle Gewichte zu ändern: kleine „Adapter"-Matrizen werden trainiert. Adapter ist 1–5% der Modellgröße. Original-Gewichte bleiben unangetastet. Beim Inference werden Adapter hinzugemerged.

  • 10× weniger GPU-RAM nötig
  • Adapter sind klein (Megabytes statt Gigabytes) — du kannst pro Use-Case einen eigenen halten
  • QLoRA = LoRA + 4-bit-Quantisierung — passt ein 70B-Modell-Tuning auf eine 24GB-Karte

3. RLHF / DPO

Statt Input-Output-Paaren: Präferenzen. „Antwort A ist besser als B." Modelle lernen aus Vergleichen. Aufwendig, aber nötig, wenn du das Verhalten trainierst (Höflichkeit, Sicherheit, Stil-Nuancen). DPO (Direct Preference Optimization) ist die handhabbarere moderne Variante.

Datenanforderungen — keine Ausreden

Wie viele Beispiele brauchst du? Mehr als du denkst, weniger als die Forschung suggeriert.

50–200 BeispieleStil, einfaches Format. Funktioniert oft schon.
500–2000Robusterer Stil, komplexere Aufgaben.
5000+Domänen-Adaptation, neue Sprache.

Qualität schlägt Quantität. 100 saubere, repräsentative, konsistente Beispiele sind besser als 10.000 halbgare. Räum dein Trainingsset auf wie deinen Kühlschrank — was riecht, fliegt.

Format des Datensatzes (OpenAI-Style)

{
  "messages": [
    {"role": "system", "content": "Du bist ein Support-Agent von Acme."},
    {"role": "user", "content": "Wo ist mein Paket?"},
    {"role": "assistant", "content": "Ich helfe dir gerne. Hast du eine Sendungsnummer?"}
  ]
}

Eine Zeile JSONL pro Trainings-Sample. Tausend Zeilen = tausend Beispiele.

Train-Routine

  1. Eval-Set zuerst. Bevor du trainierst, leg 10–20% beiseite. Die rührst du nicht an. Damit misst du, ob das Modell besser wurde — sonst optimierst du im Blindflug.
  2. Hyperparameter konservativ. Lower learning rate. Wenige Epochen (1–3). Mehr Epochs = Overfitting auf deine 200 Beispiele.
  3. Vergleich gegen Baseline. Vergleich getuntes Modell direkt gegen Basismodell mit gutem Prompt. Wenn der Prompt schon reicht, war das Tuning Verschwendung.

Cloud vs. Self-Hosted

  • OpenAI / Anthropic-FT-API — einfacher, kein GPU-Setup, monatliches Hosting kostet.
  • Hugging Face / Together / Modal — Mid-Tier-Convenience.
  • Selber auf GPU — maximale Kontrolle, RTX 4090 reicht für 7B-Modelle mit QLoRA.

Nimm den ersten Weg, der reicht. Sportler, der vom GPU-Spec angeben muss, hat keinen Volumen aufgebaut.
