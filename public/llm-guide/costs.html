Die zwölf Hebel, die wirklich Geld sparen

  1. Kleineres Modell für jede Aufgabe versuchen. 80% der Tasks reichen Haiku/4o-mini. Großes Modell nur, wenn das kleine nachweislich versagt.
  2. Prompt-Caching aktiv nutzen. Anthropic + OpenAI cachen statische Prompt-Teile. Cache-Hit kostet 10× weniger. System-Prompt + RAG-Kontext stabil halten.
  3. Batch-API. OpenAI / Anthropic / Bedrock bieten 50% Rabatt für asynchrone Batch-Calls (24h Antwortzeit). Für Backfills, Bulk-Processing.
  4. Output-Tokens minimieren. Output ist 3–5× teurer als Input. „Antworte in einem Satz" ist kein Stilmittel — es ist Kostenmanagement.
  5. System-Prompt kürzen. Jede Wiederholung einer 800-Token-Persona × 1M Calls ist ein Auto.
  6. Fewer Few-Shots. 3 gute Beispiele schlagen 10 mediocre. Kürze, was nicht beweist.
  7. RAG-Chunks knapp halten. Reranker + Top-3 statt Top-20.
  8. Streaming benutzen für UX, nicht für Speed. User wartet weniger, du zahlst gleich. Aber wahrgenommene Wartezeit halbiert.
  9. Stop-Tokens setzen. Kein Geblubber nach „---".
  10. Reasoning-Tokens deckeln. Reasoning-Modelle können dir 10k Thinking-Tokens schenken. Set max_thinking wenn die Antwort einfach ist.
  11. Function-Calling statt Freitext-Parsing. Strukturierte Outputs sparen Re-Tries.
  12. Lokal für Privates. Was nicht raus muss, läuft lokal — siehe Punk.

Beispielrechnung — sei nicht naiv

Use-Case: Chatbot mit RAG, 1000 Conversations/Tag, 5 Turns ø

Pro Turn:
  System-Prompt:    1500 Token
  RAG-Kontext:      4000 Token
  History:          1500 Token (wachsend)
  User-Input:        100 Token
  --------------------------------
  Input gesamt:     7100 Token

  Output:            300 Token

Mit Sonnet (Cache deaktiviert):
  Pro Turn = 7100 × $3/M + 300 × $15/M = $0.0258
  Pro Conversation (5 Turns) = $0.13
  Pro Tag (1000 Conv.) = $130
  Pro Monat = ~$3900

Mit Cache aktiviert (System+RAG cached):
  Cache-Hit-Cost: 5500 × $0.30/M + 1600 × $3/M + 300 × $15/M = ~$0.012
  Pro Tag = $60
  Pro Monat = ~$1800   ← ~50% gespart durch eine Setting

Wann lokal billiger wird

Faustregel: ab ~$2000/Monat Cloud-Kosten kann sich lokales Hosting lohnen — abhängig von Latenz-Anspruch, Modellgröße, Devs für Maintenance. Unter $500/Monat: Cloud, immer. Dazwischen: rechnen, ehrlich.

Eine letzte Tabelle — Output-Reduktion

Was du an den Prompt schreiben kannst, um Output kurz zu halten:

  • „Antworte in maximal X Sätzen."
  • „Keine Aufzählungen, keine Disclaimer, keine Wiederholung der Frage."
  • „Antworte nur mit der Zahl/dem Wort/dem JSON, sonst nichts."
  • „Kein vorausschauendes ‚I will now...'."

Jeder davon spart bei Skalierung dreistellig pro Monat. Das ist keine Pingeligkeit — das ist Buchhaltung.
